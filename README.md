## 🧭 Team Workflow & Commit Convention

이 프로젝트는 **Issue → Pull Request → Commit** 흐름을 명확히 정의하여 작업 단위, 변경 이력, 완료 상태를 일관되게 관리합니다.

### 🗂 Team Workflow (Issue · PR · Commit)
- 모든 작업은 **Issue 생성부터 시작**
- **1 Issue = 2 PR** 원칙
- Project는 **Issue 전용 보드**로 운영
- 이슈 종료는 **`main` 브랜치 merge 시점**에만 수행

👉 [Team Workflow 문서 바로가기](./docs/convention/workflow.md)


### 📝 Commit Convention
이 프로젝트는 **Conventional Commits** 규칙을 기반으로 커밋 메시지를 작성합니다.

자세한 커밋 규칙은 다음 문서에서 확인하시길 바랍니다.
